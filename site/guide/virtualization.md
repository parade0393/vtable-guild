# 虚拟滚动

虚拟滚动是 vtable-guild 最值得优先关注的增强能力之一。

如果你来自 ant-design-vue，这一点尤其重要，因为 ant-design-vue Table 本身并不提供虚拟滚动。对于长列表页面，这通常意味着你不再需要单独维护另一套大数据量表格方案。

## 怎么开启

虚拟滚动需要同时满足两个条件：

- virtual 为 true
- scroll.y 提供一个有效高度

```vue
<VTable
  row-key="key"
  :columns="columns"
  :data-source="dataSource"
  :scroll="{ y: 400 }"
  :virtual="true"
/>
```

只有写了 virtual 但没有 scroll.y，不会真正启用虚拟滚动。

## 适合什么场景

- 几千到上万行的数据列表。
- 需要保持表格头部、列定义和主题体系一致，不想切换成另一种列表组件。
- 页面瓶颈主要来自 DOM 数量，而不是接口响应速度。

## 使用建议

- 先在没有复杂合并单元格的列表页启用，收益最直接。
- 如果页面同时有固定列，建议先确认基础滚动体验，再逐步叠加更多复杂交互。
- rowKey 要保持稳定，否则滚动窗口复用时容易出现状态错位。

## 已知边界

- 需要配合 scroll.y 才能工作。
- 对跨行合并、复杂自定义单元格这类场景，建议先做小范围验证再全面启用。
- 虚拟滚动适合提升长列表渲染性能，但不会替你解决服务端慢查询或大对象计算问题。

## 什么时候值得优先迁移到 vtable-guild

如果你当前继续使用 ant-design-vue Table，但页面已经出现这些现象，通常就值得优先迁移：

- 滚动明显卡顿。
- 页面初次渲染过慢。
- 需要再引入额外虚拟列表方案，导致表格能力割裂。
